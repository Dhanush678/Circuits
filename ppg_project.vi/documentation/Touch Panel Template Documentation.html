Touch Panel Project

LabVIEW 2015

The Touch Panel Project template provides a starting point for creating LabVIEW touch panel applications that target an NI Touch Panel device. The design of this template makes it easy to start your touch panel project.

The Desktop Main VI of this template is based on the Queued Message Handler (QMH) template. Refer to the documentation for the QMH template for more information on how to use this design pattern.

System Requirements

  • LabVIEW Professional Development System and Application Builder
  • NI Touch Panel device: NI TPC-2206, NI-TPC 2212, NI TPC-2215, NI TPC-2230

Overview

The Touch Panel Project template includes a Touch Panel Main VI and a Desktop Main VI. These VIs communicate using shared variables. The following figure illustrates the architecture of this template. Each VI and their communication architectures are described below.

Touch Panel Main VI (Touch Panel Target)

The front panel of the Touch Panel Main VI groups controls and indicators into logical groupings using a Tab Control. You can add controls and indicators to each tab according to your application needs.

The Touch Panel Main VI is based on the Event Handler architecture. In the Timeout case of the Event Structure, the target reads the shared variables Data and Alarm and updates the indicators Data and Alarm <Replace Me>, respectively. In the Scaling <Replace Me> case, when you modify the Scaling <Replace Me> control, the control triggers an event that updates the Scaling shared variable, which the Desktop Main VI reads.

Desktop Main VI (Host Computer)

The Desktop Main VI architecture is based on the Queued Message Handler (QMH) template. Refer to the documentation for the QMH template for more information about the QMH architecture.

The primary functions of the Desktop Main VI are to generate, process, and send data. Signal generation and signal processing occur in the Timeout case of the Event Structure in the Event Handling Loop. The Random Number Generator generates data and the data is scaled according to the scaling value specified in the Touch Panel Main VI. This scaled data is sent to the Message Handling Loop using the Message Queue library.

In the Send Data case of the Message Handling Loop, the host computer processes the data from the Event Handling Loop. This value transfers to the target via the Data shared variable. This data is also compared to the Alarm Threshold <Replace Me> control, and the alarm status writes to the Alarm shared variable, which is read on the target.

Running this Template

  1. In the Project Explorer window, under My Computer, open and run Desktop Main.vi.
  2. Under the touch panel target, open and run Touch Panel Main.vi. On the Process Control tab, specify the Scaling <Replace Me> control. On the touch panel device, watch the Touch Panel Main.vi running.
  3. On the host computer, switch to Desktop Main.vi and specify the Alarm Threshold <Replace Me> control. Watch the indicators on the different tabs of Touch Panel Main.vi display data that corresponds to data from Desktop Main.vi.

Modifying this Template

The Touch Panel Project template illustrates general function and communication strategy. You can modify the following elements of this template to suit your application.

Touch Panel Main VI (Touch Panel Target)

Front Panel

You can remove or rename the controls and indicators labeled with <Replace Me> and add your own controls and indicators to the tab control. You may need to rename the tabs and adjust the number of tabs, according to the groupings required for your application.

Block Diagram

Due to the limited processing power of touch panel computers, National Instruments recommends that you offload computation to the desktop or other more powerful targets whenever possible. To customize the block diagram, you can delete or rename the existing shared variables to fit your application. You might need to create an Event case for each of the controls you add to the front panel. As with all event structures, keep the code associated with each case to a minimum. Shared variables can be read in the Timeout case of the Event Structure.

Desktop Main VI (Host Computer)

Front Panel

You can update or remove the Alarm Threshold <Replace Me> control. You also can add the controls and indicators for your application needs.

Block Diagram

You can modify the data generation and data processing from the Timeout case in the Event Handling Loop with your own data generation or acquisition code. If you have minimal data processing, you can retain the existing data processing in the Timeout case. If, however, you have more computationally intensive application needs, move the data processing to the Message Handling Loop. To move your processing to the Message Handling Loop, add a dedicated data processing case to the Case Structure.

In the Send Data case of the Message Handling Loop, you can remove or modify the alarming functionality to suit your application.

Note: The data transfer in this template is lossy. For lossless data transmission, National Instruments recommends that you use another data transfer protocol, such as Network Streams.
